# Vim Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add toggleable vim keybindings to all Monaco editor instances (FileEditor and DiffEditor) via the `monaco-vim` library, with a global setting persisted across app restarts.

**Architecture:** A Zustand store (`useEditorSettingsStore`) holds the `vimEnabled` boolean. It's persisted through the existing `collectSnapshot()`/`hydrate()` session save cycle in `App.tsx`. Each Monaco editor subscribes to the store and dynamically attaches/detaches `monaco-vim` with a status bar rendered below the editor.

**Tech Stack:** `monaco-vim`, Zustand, Monaco Editor, existing sessionState persistence

**Spec:** `docs/superpowers/specs/2026-03-22-vim-mode-design.md`

- [ ] **Step 3: Write the store implementation**

Create `src/renderer/stores/editor-settings.ts`:

```typescript
import { create } from "zustand";

interface EditorSettingsStore {
	vimEnabled: boolean;
	setVimEnabled: (enabled: boolean) => void;
	hydrateVimMode: (value: string | undefined) => void;
}

export const useEditorSettingsStore = create<EditorSettingsStore>((set) => ({
	vimEnabled: false,
	setVimEnabled: (enabled) => set({ vimEnabled: enabled }),
	hydrateVimMode: (value) => set({ vimEnabled: value === "true" }),
}));
```

	const editor = editorRef.current;

	if (vimEnabled && vimStatusRef.current) {

		vimModeRef.current = null;
	};
