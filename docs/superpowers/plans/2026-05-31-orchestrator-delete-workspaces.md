# Orchestrator Delete-Its-Workspaces Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** When deleting a cross-repo orchestrator, optionally also permanently remove the worktree workspaces it dispatched (force, including uncommitted changes), via an explicit second confirmation.

**Architecture:** A new boolean column `created_by_dispatch` on `orchestrator_members` marks members created by `dispatchAcrossRepos`. `deleteCrossRepoOrchestrator` gains an opt-in `removeWorkspaces` flag that calls the existing `removeWorkspace({ force: true })` for each dispatched member before the existing orchestrator cleanup. The renderer asks two `window.confirm` questions: delete the orchestrator, then (only if it created workspaces) keep-or-delete those workspaces.

**Tech Stack:** Electron + React 19 + TypeScript, Bun (runtime + test), Drizzle ORM/SQLite, tRPC over Electron IPC, Biome.

**Conventions for every task below:**
- Run all commands from `apps/desktop/`.
- Run a single test file with `bun test tests/<file>.test.ts`.
- Format/lint touched files with `bunx biome check --write <paths>` before committing.
- Tabs for indentation, double quotes, semicolons, line width 100.
- Commit messages: NO `Co-Authored-By` trailer. Never use `--no-verify`.

```ts
export async function deleteCrossRepoOrchestrator(input: {
	id: string;
	removeWorkspaces?: boolean;
}): Promise<{ ok: true }> {
	const row = await getCrossRepoOrchestrator({ id: input.id });
	if (!row) return { ok: true };

	if (input.removeWorkspaces) {
		// Capture the workspace ids the orchestrator created before deleting anything,
		// then force-remove each. `force` skips the uncommitted-changes check (the user
		// consented). Wrapped per-workspace so one failure cannot abort the rest or the
		// orchestrator deletion. Dispatched workspaces are always type "worktree", so the
		// "cannot delete the main branch workspace" guard in removeWorkspace never trips.
		const dispatched = getDb()
			.select({ workspaceId: orchestratorMembers.workspaceId })
			.from(orchestratorMembers)
			.where(
				and(
					eq(orchestratorMembers.orchestratorId, input.id),
					eq(orchestratorMembers.parentKind, "cross_repo"),
					eq(orchestratorMembers.createdByDispatch, true)
				)
			)
			.all();
		for (const m of dispatched) {
			try {
				await removeWorkspace({ workspaceId: m.workspaceId, force: true });
			} catch (err) {
				console.warn("[xro] removeWorkspace failed during orchestrator delete:", err);
			}
		}
	}

	try {
		rmSync(row.workDir, { recursive: true, force: true });
	} catch {}
	getDb()
		.delete(orchestratorMembers)
		.where(
			and(
				eq(orchestratorMembers.orchestratorId, input.id),
				eq(orchestratorMembers.parentKind, "cross_repo")
			)
		)
		.run();
	getDb().delete(crossRepoOrchestrators).where(eq(crossRepoOrchestrators.id, input.id)).run();
	removeCrossRepoEventsFile(input.id);
	invalidateAllCrossRepoLinks();
	return { ok: true };
}
```
